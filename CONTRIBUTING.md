# Contributing to AletheIA

Thank you for contributing to AletheIA.

## What to prioritize

Please prioritize contributions that improve:

- clarity
- determinism
- explainability
- portability
- governance
- quality baseline
- starter-pack reuse

## Before opening changes

Please make sure that:

- the change has a clear scope
- examples remain small and readable
- tests still pass
- public docs stay understandable for non-specialists

## Good first contribution areas

- documentation clarity
- examples
- schemas
- deterministic tests
- starter-pack guides and templates

## Avoid by default

- large hidden abstractions
- provider-specific assumptions in the core
- changes that make the framework harder to inspect
