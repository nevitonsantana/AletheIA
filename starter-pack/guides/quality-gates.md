# Quality Gates Guide

AletheIA assumes that relevant work should not be concluded without proof.

## Minimum proof can include

- tests
- smoke checks
- lint or static validation
- policy evaluation
- functional verification

## Proportionality

Not every task needs the same level of proof.

But every relevant task needs some proof.

## Closure rule

If validation is missing, closure should slow down, stop, or produce a learning for future prevention.
